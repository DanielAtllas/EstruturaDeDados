Introdução - Estrutura de Dados

Ao desenvolver um programa para realizar o processamento de dados, é preciso transcrever de forma que o computador possa compreender e executar tal programa e que o programador também compreenda o que escreveu. As linguagens de programação são códigos esritos em uma linguagem que o programador compreende e que o computador consegue interpretar e executar.

As estruturas de dados mais simples e que oram estudadas na disciplinas de Programação Estruturada são os vetores e matrizes. Estas estruturas de dados são estruturas de ddos homogêneos, pois permitem o armazenamento de dados de um único tipo de dado. Elas permitem acesso direto a um elemento através do nome vetor/matriz seguido do índice. Permitem também o acesso sequêncial, percorrendo elemento a elemento do vetor/matriz.

Entretanto, existem outras estruturas de dados mais complexas e que são usadas para representar de forma organizada arranjos de dados específicos. Sendo as seguintes estruturas:

  • Lista

    • Uma lista é uma estrutura de dados composta por nós e elementos, que apontam para o próximo elemento da lista, fazendo com que o seu último elemento aponte para NULL, ou seja, a lista será compasta por elementos/nós, onde cada elemento/nó possuirá um ponteiro apontador para o elemento seguinte e assim sucetivamente.

  • Pilha

    • As filas são estruturas de dados baseadas na idéia de que o primeiro elemento a entrar na pilha, é o primeiro elemento a sair. Esta ideia é conhecida como FIFO (first in, first out). Uma fila possui duas funções básicas, sendo de adição no final da pilha e remoção do início da pilha.

  • Árvore

    • Uma árvore é uma estrutura de dados em que cada elemento tem um ou mais elementos associados. Uma árvore possui apenas uma raiz. Em uma árvore, os elementos associados a cada nó, são habitualmente chamaos de filhos (seguindo a ideia de galhos de uma árvore).

  • Grafo

    • Um grafo é uma estrutura de dados composta por um conjunto de vértices e arestas. Sendo que cada aresta é representada por um par de vértices.

Elementos fundamentais das estrutuas de dados

Estas estruturas de dados fazem uso de tipos abstratos de dados, alocação dinâmica de memória e ponteiros que são os elementos fundamentais a partir dos quais são construídas estas estruturas.

  • Ponteiros são usados para referênciar os elementos da estrutura de dados e permitir o encadeamento dos elementos, a ligação entre os elementos que compôem um estrutura.

  • Tipos abstratos de dados são usados para representar elementos das estruturasque podem ser compostos por atributos de tipos primitivos ou por atributos de outros tipos abstratos de dados.

  • Alocação dinâmica é usada para permitir a criação de novos elementos da estrutura de dados em tempo de execução.

O uso destas estruturas de dados em alguns casos é a única forma de se representar os dados a serem processadas em um determinado problema. Em outros casos, os dados poderima ter sido processados de maneira diferente, porém o programa não seria eficiente (alto consumo de memória) ou não teria um bom desempenho.
